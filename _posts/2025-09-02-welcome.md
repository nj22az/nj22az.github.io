This guide shows you how to create, format, and publish blog posts on your Jekyll + React website. All content is managed through Markdown files with powerful customization options.

## Creating a New Post

### File Naming Convention
Create new posts in the `_posts/` directory using this naming pattern:
```
YYYY-MM-DD-post-slug.md
```

**Examples:**
- `2025-01-15-getting-started.md`
- `2025-02-20-advanced-configuration.md`
- `2025-03-05-troubleshooting-guide.md`

### Basic Post Structure

Every post starts with YAML front matter containing metadata:

```yaml
---
layout: post
title: "Your Post Title"
date: 2025-01-15
categories: [Tutorial]
tags:
  - Guide
  - Setup
  - Documentation
content_type: Documentation
featured: false
cover_icon: file-text
thumbnail: /assets/images/posts/post-slug/preview.jpg
---

Your post content goes here in Markdown format.
```

## Available Icons

Choose from over 30 Lucide icons for your posts. Add `cover_icon: icon-name` to your front matter:

**Writing & Content:**
- `notebook-pen` - Journal entries, personal reflections
- `text-quote` - Quotes, insights, philosophical content
- `pencil` - Writing guides, tutorials
- `newspaper` - News, announcements, updates

**Technology:**
- `code` - Programming, development guides
- `terminal` - Command line tutorials
- `laptop` - Tech reviews, software recommendations
- `server` - Infrastructure, hosting guides

**Documentation:**
- `file-text` - Detailed guides, documentation
- `clipboard` - Checklists, step-by-step processes
- `folder` - Resource collections, curated lists
- `book` - Learning materials, educational content

[View all available icons in `_data/icons.yml`](/_data/icons.yml)

## Adding Images

### Image Organization
Store post images in organized folders:
```
assets/images/posts/post-slug/
├── hero-image.jpg
├── diagram.svg
├── screenshot.png
└── gallery/
    ├── photo1.jpg
    └── photo2.jpg
```

### Image Syntax
Reference images using site-relative paths:

```markdown
![Alt text](/assets/images/posts/post-slug/image.jpg)

<!-- With caption -->
![Configuration example](/assets/images/posts/how-to-write-posts/markdown-example.svg)
*Example of Markdown syntax formatting*
```

### Image Best Practices
- Use descriptive file names: `config-panel-settings.jpg` not `image1.jpg`
- Optimize images for web (compress large photos)
- Use SVG for diagrams and illustrations
- Include alt text for accessibility

## Markdown Formatting

### Headings
```markdown
# Main Title (H1) - Used automatically from front matter
## Section Heading (H2)
### Subsection (H3)
#### Minor Heading (H4)
```

### Text Formatting
```markdown
**Bold text** for emphasis
*Italic text* for subtle emphasis
`Code snippets` for technical terms
~~Strikethrough~~ for corrections
```

### Lists
```markdown
Unordered lists:
- First item
- Second item
  - Nested item
  - Another nested item

Ordered lists:
1. First step
2. Second step
3. Third step
```

### Links
```markdown
[Link text](https://example.com)
[Internal link](/about)
[Link to another post](/2025/01/15/other-post)
```

### Code Blocks
````markdown
```bash
# Terminal commands
jekyll serve --livereload
```

```yaml
# YAML configuration
title: "Example Post"
cover_icon: code
```

```javascript
// JavaScript example
function hello() {
  console.log("Hello, world!");
}
```
````

### Blockquotes
```markdown
> Important information or quotes
> can span multiple lines
>
> Start new paragraphs with a blank line
```

## Publishing Workflow

### 1. Create the Post
```bash
# Create new post file
touch _posts/2025-01-15-my-new-post.md

# Create image directory
mkdir -p assets/images/posts/my-new-post
```

### 2. Add Content
Write your post content using the examples above.

### 3. Preview Locally
```bash
# Start development server
jekyll serve --livereload

# View at http://127.0.0.1:4000
```

### 4. Commit and Deploy
```bash
# Add all changes
git add -A

# Commit with descriptive message
git commit -m "Add new post: How to configure settings"

# Deploy to GitHub Pages
git push origin main
```

## Advanced Features

### Featured Posts
Set `featured: true` in front matter to display the post prominently on the homepage.

### Custom Thumbnails
Add `thumbnail: /path/to/image.jpg` for custom preview images in post cards.

### Content Types
Use `content_type` to categorize posts:
- Documentation
- Tutorial
- Guide
- Review
- Analysis

### Tags and Categories
```yaml
categories: [Tutorial, Advanced]
tags:
  - Configuration
  - Setup
  - Advanced
  - Jekyll
```

Tags appear as clickable pills on post cards and can be used for filtering.
